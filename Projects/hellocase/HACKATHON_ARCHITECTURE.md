# 黑客松 MVP 技术架构规划

## 竞品分析 & 对标

### 竞品 1：Notion
- **架构**：云端 + React 前端 + Node.js 后端 + 数据库
- **核心流程**：输入 → 实时同步云端 → 数据库存储 → 前端渲染
- **优点**：功能强大、云端同步
- **缺点**：复杂度高、学习成本大
- **黑客松相关性**：❌ 太复杂，不适合半天完成

### 竞品 2：Obsidian
- **架构**：本地 Markdown 文件 + Electron/Web 前端 + 本地图谱算法
- **核心流程**：写 MD 文件 → 解析链接 → 生成知识图谱 → 前端展示
- **优点**：简洁、隐私保护、无需后端
- **缺点**：没有 AI 能力
- **黑客松相关性**：✅ 架构简洁，但缺少 AI 这是我们的创新点

### 竞品 3：Get 笔记（得到）
- **架构**：云端知识库 + AI 生成摘要 + 订阅制
- **核心流程**：输入 → LLM 生成摘要 + 分类 → 展示
- **优点**：有 AI 能力、UX 好
- **缺点**：不开源、商业产品
- **黑客松相关性**：✅ 这是我们的最直接对标

---

## 我们的黑客松架构（超精简版）

### 🎯 设计原则
1. **快速上线**：放弃复杂功能，只做核心体验
2. **无需数据库**：用 JSON 或内存存储（5 小时没时间写 SQL）
3. **单一职责**：前端 + 后端 + AI，各自独立
4. **演示友好**：能看到实时效果的产品

## 数据流（用户的角度）

```
步骤 1：用户粘贴内容
  输入：「小红书笔记：如何设计一个完美的钱包？」

步骤 2：前端 Streamlit 调用 analyze_one()
  POST → Anthropic API
  Prompt = analyze.txt 模板 + 用户内容

步骤 3：Claude 返回 JSON
  {
    "summary": "...",
    "tags": ["设计", "消费心理", "产品创新"],
    "keywords": ["钱包", "用户体验", "心理学"],
    "category": "社媒灵感",
    "insight": "..."
  }

步骤 4：保存到 JSON + 渲染卡片
  add_insight() → insights.json
  render_result_card() → 前端展示
```

## 核心组件

### 1. Streamlit 单文件应用（app.py，约 280 行）
- **数据层**：load/save_insights / add_insight
- **AI 层**：analyze_one / cluster_insights
- **可视化层**：render_heatmap / render_tag_cloud
- **UI 层**：sidebar / 3 个 Tab（捕获/可视化/洞察）

### 2. Prompts（外部文件，方便迭代）
- `prompts/analyze.txt`：单条灵感分析
- `prompts/cluster.txt`：跨灵感主题发现

### 3. 数据存储（无数据库）
- `data/insights.json`：所有灵感
- 含 3 条预置 demo 数据（穿搭 / 产品会议 / 学习笔记）

### 4. 配置
- `.env`：ANTHROPIC_API_KEY
- `requirements.txt`：streamlit / anthropic / python-dotenv / plotly / pandas

## 技术栈总结

| 层级 | 技术 | 为什么 |
|------|------|----------|
| **前端** | Streamlit | 30 分钟出前端 |
| **后端** | Streamlit（同进程）| 不需要分离 |
| **AI** | Claude Haiku | 中文理解最好 + 快 + 便宜 |
| **存储** | JSON 文件 | 黑客松不需要数据库 |
| **可视化** | Plotly | 现成的热力图组件 |
