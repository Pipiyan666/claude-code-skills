## 问题

本地 95 个 + 在线 91,666+ = 90,000+ skills，覆盖大多数场景。
但仍有**超难度任务**无现成 skill 可用的情况。

## 解决方案：五层递进策略

### Layer 1: 本地库查询 ✅
- 路径：`/Users/pipipiyan/everything-claude-code/skills/`
- 时间：快速（本地）
- 成功率：~70% 任务能找到

### Layer 2: 在线库查询 ✅
- URL：https://skills.sh/
- 时间：中等（网络访问）
- 成功率：~20% 额外任务能找到
- **累计成功率：~90%**

### Layer 3: 组合多个 Skills ✅
对于不能单个 skill 覆盖的任务：
- 识别任务的多个维度
- 查找多个相关 skills
- 组合它们的最佳实践
- 例如：写 TypeScript 的安全代码 = `typescript-lsp` + `security-review` + `coding-standards`

**什么时候用**：
- 任务涉及多个领域（如"设计一个安全的 API"）
- 需要混合不同技术栈（如"用 Go + PostgreSQL 做并发处理"）
- 需要完整工作流（如"从设计到测试到部署"）

### Layer 4: 深度研究模式 🔬
当 Layer 1-3 都不够时，启动"研究模式"：

**做法**：
1. **分解任务** - 把超难度任务拆成多个小问题
2. **逐一查询** - 每个小问题都尝试 Layer 1-3
3. **组合知识** - 把这些小知识综合成完整方案
4. **记录过程** - 记录整个研究和决策过程

**例子**：
```
任务：实现一个分布式事务管理系统
  ↓
分解为：
  - 分布式一致性问题 (Layer 1: backend-patterns)
  - 数据库事务设计 (Layer 2: postgres-patterns)
  - 错误恢复机制 (Layer 3: 组合多个 skills)
  - 测试策略 (Layer 1: tdd-workflow)
  ↓
综合这些知识设计方案
```

### Layer 5: 创建新 Skill（终极方案）💡

当任务完全创新、无现成参考时：

**做法**：
1. **完成任务** - 用最好的工程实践，即使没有 skill 参考
2. **总结学到的** - 记录整个过程和决策
3. **提炼最佳实践** - 哪些做法有效？为什么？
4. **创建新 Skill** - 把知识编织成可重用的 skill
5. **共享到库** - 保存到本地库

**这是最有价值的！** 因为：
- 你会成为这个领域的专家
- 未来遇到类似问题直接有 skill
- 可以分享给其他人

## 难度分类

### 🟢 简单任务 (70% 出现)
- 有现成的本地 skill
- 按 skill 做就行
- 时间：快速

### 🟡 中等任务 (20% 出现)
- 在线库有相关 skill
- 或需要组合多个 skills
- 时间：中等
- 我需要做额外研究

### 🔴 超难度任务 (10% 出现)
- 完全创新的领域
- 无现成最佳实践
- 需要深度研究 + 创新思考
- **时间：较长，但产出最有价值**

## 超难度任务的完整流程

```
用户给任务
  ↓
我评估难度（简单/中等/超难？）
  ↓
简单？ → 用本地 skill，快速完成 ✅
  ↓
中等？ → 查在线库 + 组合 skills，完成 ✅
  ↓
超难？ → 进入研究模式
  ↓
  1. 分解任务为多个子问题
  2. 逐一查询 layers 1-3
  3. 进行深度研究和创新思考
  4. 综合知识设计方案
  5. 执行并记录过程
  ↓
完成任务
  ↓
创建新 skill（最重要！）
  ↓
保存到库
  ↓
下次遇到类似任务就轻松了
```

4. ✅ **完整记录** - 把整个过程记录下来
5. ✅ **创建 skill** - 把知识沉淀为可重用的 skill

## 实际例子

### 例子 1：简单任务
**任务**："用 Python 写个数据处理脚本"
- 难度：🟢 简单
- 处理：查 `python-patterns` → 套用最佳实践 → 完成
- 时间：30 分钟

### 例子 2：中等任务
**任务**："设计一个实时协作编辑的 WebSocket API"
- 难度：🟡 中等
- 处理：
  - 查 `api-design` (API 架构)
  - 查 `backend-patterns` (并发处理)
  - 组合：WebSocket + 并发控制 + 冲突解决
- 时间：2 小时

### 例子 3：超难度任务
**任务**："实现一个 AI 驱动的代码审查系统，能理解上下文、学习团队风格、给出智能建议"
- 难度：🔴 超难
- 处理：
  1. 分解：AI 模型选择 / 代码解析 / 上下文提取 / 风格学习 / 建议生成
  2. Layer 1-3 查询：`security-review` / `code-review` / 等
  3. 深度研究：AI 如何理解代码、如何学习团队风格
  4. 创新设计：把这些结合成一个完整系统
  5. 创建新 skill：`ai-code-review-system`
- 时间：1-2 天，但产出极有价值
